# Ballhex UI Art Prompt Library

## 1. Purpose

This document is a visual prompt library for generating UI mockups and reusable art assets for Ballhex.

Use these prompts in another AI image tool to generate:

- Full screen mockups
- UI exploration boards
- Background illustrations
- Reusable decorative assets
- Avatar sets
- Icons and badges

The prompts are written in English because most image models respond better that way.

## 2. Product Direction

Ballhex is:

- a 2D football game
- arcade, competitive, social
- mobile-first in visual language
- cartoon, energetic, premium

The UI should feel like:

- a modern mobile multiplayer sports game
- clean and readable
- bold but not childish
- layered and animated in spirit

Avoid:

- generic esports dashboard look
- flat gray debug-tool panels
- realistic football broadcast UI
- overly dark cyberpunk style
- purple-heavy fantasy palette

## 3. Global Visual Style Prompt

Use this as the base style prompt before every screen prompt.

### Base prompt

`A premium mobile arcade football game interface, cartoon stylized but polished, modern sports game UI, bold rounded cards, glossy glassmorphism surfaces, dark navy to deep green gradient background, subtle stadium atmosphere, vivid coral red and electric blue team accents, soft neon lime highlights, clean hierarchy, readable typography areas, premium game menu presentation, layered panels, subtle glow, subtle shadows, playful but competitive, polished mobile game UX, football themed shapes, hexagon motifs, dynamic composition`

### Negative prompt

`ugly layout, low contrast, generic admin dashboard, bland spreadsheet UI, overly realistic football broadcast graphics, sci-fi spaceship UI, fantasy RPG UI, purple dominant palette, cluttered screen, unreadable text, noisy background, cheap mobile game ad style, low quality buttons, poor spacing, flat boring cards`

## 4. Output Guidelines

Ask the image model for these variations when possible:

- `16:9 landscape` for gameplay and desktop/web view
- `9:16 portrait` for mobile-first menu ideation
- `clean UI concept with empty text placeholders`
- `high fidelity game UI mockup`
- `separate asset sheet on transparent background` when needed

If you want editable layering references, ask for:

- `clear panel separation`
- `distinct button groups`
- `clean background with isolated decorative elements`

## 7. Recommended Production Order

1. Background pack
2. Button pack
3. Card/panel pack
4. Main menu mockup
5. Custom lobby browser mockup
6. ESC room overlay mockup
7. Match HUD and chat mockup
8. Profile setup mockup
9. Ranked menu mockup
10. Avatar pack
11. Badge pack
12. Icon pack

## 8. Best Workflow With Me

You can use this document in a loop:

1. Generate one screen or one asset pack.
2. Send me the result.
3. I compare it against the game layout and tell you:
   - what fits
   - what is too noisy
   - what should be simplified for Godot
   - what can become reusable UI elements
4. Then we turn the chosen direction into actual scenes and UI assets.
